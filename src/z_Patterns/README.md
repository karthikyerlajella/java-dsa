# 🧩 DSA Patterns Revision Hub

This section contains the most important problem-solving patterns used in coding interviews.

Unlike topic-based folders (Arrays, Linked Lists, Trees, etc.), this folder focuses on **how a problem is solved**, not where it comes from.

Many interview questions can belong to different topics but share the same underlying pattern.

Example:

* Two Sum II → Arrays + Two Pointers
* Longest Substring Without Repeating Characters → Strings + Sliding Window
* Linked List Cycle → Linked List + Fast & Slow Pointer

The goal of this section is to build strong pattern recognition and improve interview problem-solving speed.

## 🎯 How To Use This Section

Each pattern file contains:

* Pattern Definition
* When to Use
* Recognition Clues
* Generic Template
* Common Mistakes
* Related Problems

This folder is intended for revision and interview preparation.

Actual problem solutions are stored in their respective topic folders:

```text
c_Arrays/
d_Strings/
e_LinkedList/
...
```

This avoids duplicate code and keeps the repository organized.

---

## 🧠 Interview Mindset

When solving a problem, ask:

1. Which topic does this belong to?
2. Which pattern can solve it efficiently?

Examples:

| Problem                                        | Topic       | Pattern              |
| ---------------------------------------------- | ----------- | -------------------- |
| Two Sum II                                     | Arrays      | Two Pointers         |
| Container With Most Water                      | Arrays      | Two Pointers         |
| Longest Substring Without Repeating Characters | Strings     | Sliding Window       |
| Linked List Cycle                              | Linked List | Fast & Slow Pointer  |
| Subarray Sum Equals K                          | Arrays      | Prefix Sum + HashMap |

---

## 🚀 Goal

The objective is not to memorize solutions.

The objective is to recognize patterns quickly and apply them to unseen interview problems.

Pattern recognition is one of the most valuable skills in Data Structures & Algorithms interviews.
